3. Ausgangssituation

Der Josephusring ist eine von zahllosen verschiedenen Bezeichnungen für einen einfachen Auszähl(-reim), in der ein Auswahlmechanismus auftaucht, den wohl schon jeder als Kind praktiziert hat:

Vor langer Zeit als die Sitten noch rau wahren, wurden Josephus und eine Anzahl weiterer Gefangener zu einer schlimmen Strafe verurteilt. Nachdem aber an diesem Tag der König Geburtstag hatte, sollte einer der Verurteilten zum Ruhme des Herrschers freigesprochen werden. Um eine zufällige Auswahl zu treffen, mussten die Gefangenen sich im Kreis aufstellen. Nun wurde, beginnend mit dem ersten Gefangenen, reihum von 1 bis zu einer vorher festgelegten "fatalen Zahl" abgezählt. Wen die fatale Zahl traf, an dem wurde das Urteil sofort vollstreckt und das Abzählen mit dem nächsten Delinquenten neu begonnen. Der Letzte, der übrig blieb, wurde schließlich begnadigt.

    • uuid:

    • Der Ring darf aus max. 25 Personen bestehen

5.4.1. Methode initRing(int noOfPersons)

  • Diese Methodeerstellt die Knoten (Node) mit ihren zugehörigen Personen.

  • Die Personen werden der Einfachheit halber nur mit Buchstaben aufsteigend benannt

  • Jeder Knoten hat eine Positionsnummer

  • Die Id des Knotens ist hier noch nicht relevant, da die Id erst für die DB gebraucht wird.

Figure 1. Object Diagram - Node und Person

  • Hier ist die noOfPersons = 4

  • Erstellen Sie so viele Knoten wie Personen und verknüpfen sie sie wie in diesem Objektdiagramm angegeben.

Verwenden Sie zusätzlich zur Referenzvariable head noch eine Referenzvariable current

5.4.2. Methode findLastPerson()

  • Hier wird der Josephusring so lange ausgezählt, bis nur mehr eine Person übrig bleibt

A (0) - B (1) - C (2) - D (3)
A (0) - B (1) - D (3)
A (0) - D (3)
A (0)

  1. Die fatale Zahl ist 3

  2. Man zählt: eins - zwei - C ist weg

  3. Man setzt bei D fort

  4. eins - zwei - B ist weg

  5. man setzt bei D fort

  6. eins - zwei - D ist weg

  7. A bleibt übrig

Figure 2. Object Diagram - Abbruchbedingung

  • Abbruchbedingung: Der letzte Knoten zeigt auf sich selbst

5.4.3. Methode toString()

  • gibt den Ring als String zurück

  • In den Klammern sind die Positionsnummern

  • Die Buchstaben sind die Namen der Personen

z.B.
A (0) - B (1) - C (2) - D (3)

  • Diese Methode ist sehr gut geeignet, um zu überprüfen, ob der Algorithmus funktioniert.

5.4.4. Methode addLineToOutput(String line)

  • fügt dem StringBuilder output eine Zeile mit dem momentanen Zustand des Rings hinzu.

  • Wird von der Methode findLastPerson() verwendet
